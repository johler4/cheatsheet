What is HTML?

HTML stands for Hyper Text Markup Language. HTML describes the structure of a Web page and is the standard markup language for creating web pages. It consists of a series of elements that tell the browser how to display the content. HTML elements label pieces of content such as 'this is a heading', 'this is a paragraph', 'this is a link', etc.

HTML Boiler Plate

  1. < !DOCTYPE html >
  2. < html lang="en" >
  3. < head >
  4. < meta charset="utf-8" >
  5. < meta name="viewport" content="width=device-width, initial-scale=1.0" >
  6. < title > Page Title < /title >
  7. < link rel="stylesheet" href="#" >
  8. < /head >
  9. < body >
  10. < /body>
  11. < / html >

Boiler Plate Explained:

  1. Line 1 is the document type declaration. All HTML documents must start with this. A doctype tells the broswer what version of HTML the page is written in. If this is forgotten in your code file, then some HTML5 tags (like < article > and < footer >) may not be read by the browser.
  2. Line 2 is your root element. The < html > tag is the top level element of the HTML file. You will nest the < head > and < body > tags inside of it. The 'lang=""' attribute inside the opening < html > tag set the language for the page. It is also good to include it for accessibility reasons, beacause screen readers will know how to properly pronounce the text.
  3. Line 3 begins our < head > section. Our < head > section contains information that is processed by machines. Inside the < head > tags you will nest metadata which is data that describes the document to the machine.
  4. Meta tags define metadata about an HTML document. Meta data is used by browsers (how to display content or reload page), search engines (keywords), and other webs services. Our first < meta > tag is being used to define our character set. UTF-8 (can be written in either lower or upper-case) is the standard character encoding you should use in your web pages. UTF-8 stands for Unicode Transformation Format 8-bit and is the worlds most common character encoding.
  5. Our next < meta > tag on line 5 defines the size of the viewport and renders the width of the page to the width of the device's screen. If you have a mobile device that is 600px wide, then the browser window will also be 600px wide. The "intial-scale=1.0" controls the zoom level. The value of the 1 for initial-scale prevents the default zoom by browsers.
  6. Line 6 is our < title > tag. The text put in these tags is the title for our web page and is what is shown in the browser's title bar. This text is extremely important as it is critical for SEO (search engine optimization) and affects how Google's algorithms understand your content and rank your page.
  7. < link > tags link other external sources to the current document. The < link > tag is most often used to link to external style sheets or to add a favicon or font-style to your website. On this line of our boilerplate we are linking to an external stylesheet. The "type="stylesheet" attribute tells the link tag that the "href=''" attribute is a CSS stylesheet. Inside of the href attribute you will need to list the location of your external stylesheet.
  8. Line 8 is our closing tag for the < /head > section.
  9. Line 9 is our opening tag for our < body >. The < body > defines the body of our document. This section will contain all the contents of our HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, nav's, footers, etc. There can only be one < body > element.
  10. Line 10 is our closing < /body > tag.
  11. Line 11 is our closnig < /html > tag

HTML Elements

An HTML element is a component of an HTML document that tells a web browser how to structure and interpret a part of the HTML document. An element is defined by a start tag, some content, and an end tag:

< tagname > Content goes here < /tagname >

Some examples of elements are:

Element Name
< a > < /a > Anchor (creates a link)
< br > Defines a single line break (does not require a closing tag)
< button > < /button > Defines a clickable button
< div > < /div > Defines a section in a document
< footer > < /footer > Defines a footer for a document or section
< h1 > < /h1 > Heading (h1-h6)
< header > < /header > Defines a header for a document or section
< p > < /p > Paragraph
< img > Image tag (does not require a closing tag)

For a full list of HTML elements visit this W3 Schools Link

HTML Attributes

An HTML attribute is a piece of markup language used to adjust the behavior or display of an HTML element. For example, attributes can be used to change the color, size, or functionality of HTML elements. All HTML elements can have attributes, but not all attributes can be used for every element. When asigning an attribute to an element, the attribute is always specified within the start tag and usually come in a name/value pair like name="value".

href Attribute

Often used with the < a > tag. The href attribute specifies the URL of the page the link goes to:

< a href ="https://www.google.com" >This would link to Google< /a >

src Attribute

Often used with the < img > tag. When used in an img element the src attribute embeds an image in an HTML page. The src attribute specifies the path to the image to be displayed. The path can either be internal or external (local or from the Web):

< img src="imagesfolder/image.jpeg" >

The above image that would show would be the file titled 'image.jpeg' located in the directory titled 'imagesfolder' on your local machine. Make sure when linking locally that you are creating a direct path from the index.html file that you are embedding the image to! Images from the web will need the full URL for the image to be displayed.

width and height Attributes

These two attributes can be used to define the size of various elements including divs, headers, footers, navs, images, videos, etc.:

width: 100px;
height: 100px;

alt Attribute

The alt attribute specifies an alternative text for an image if the image for some reason cannot be displayed. This is required on all "img" elements so that if the image is not loaded, the alternate text will be displayed. This is also used for those people who use screen readers so that the image alt text is then read to them to describe the image. Alt text should be breif but descriptive:

< img href="butterfly.jpeg" alt="a beautiful butterfly" >

style Attribute

The style attribute is used to add CSS styling to a page or individually to each element. Inline styling is not good practice as is can very quickly clutter up an HTML document. Styling within the head element, even though cleaner than inline-styling, is also not good practice. Styling should always be done on a seperate style.css file and should be linked through the head element using a link tag.

Lists and Tables

Unordered List

Unordered lists will show lists of items with a default list style of bullet points. The list style can be changed or removed by using the list-style=" " attribute. The opening and closing tags are "ul" and each line item listed between the two tags would be contained within opening and closing "li" tags. Example code:

< ul >
< li > < /li >
< li > < /li >
< li > < /li >
< /ul >

Ordered Lists

Ordered lists are similiar to unordered lists, but rather than using bullet points, ordered lists use numbers for their list items. Ordered lists begin and end with "ol" tags and use the same "li" tags for the list items. Example code:

< ol >
< li > < /li >
< li > < /li >
< li > < /li >
< /ol >

Tables

Tables can be used to put various pieces of information into a table format. A table is defined using beginning and ending "table" tags. Within those tags you will have nested "thead" and "tbody" tags to separate or table head and table body. Within your table head and table body you will have "tr" tags that define your table rows. And then nested in your table rows you will have "tr" tags for your table head, and "td" tags for your table body. These "th" and "td" tags are what actually contain the information that you want to put in each cell of the table. Example code:

< table >
< thead >
< tr >
< th > Table head content row 1 col 1 < /th >
< th > Table head content row 1 col 2 < /th >
< /tr >
< /thead >
< tbody >
< tr >
< td >table body content row 1 col 1 < /td >
< td >table body content row 1 col 2 < /td >
< /tr >
< tr >
< td >table body content row 2 col 1 < /td >
< td >table body content row 2 col 2 < /td >
< /tr >
< /tbody >
< /table >

The above code would should a table with three rows (one head and two body) and two columns. It would look something like this:

head1 head2
body col1 row1 body col2 row1
body col1 row2 body col2 row2
